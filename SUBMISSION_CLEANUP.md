## 🔴 SECURITY — fix before sharing the repo publicly

**`backend/esp32/src/main.cpp` contains plaintext secrets, and they're in git history:**

```c
#define WIFI_SSID  "<redacted>"
#define WIFI_PASS  "<redacted>"
#define API_KEY    "<redacted — rotate this>"
```

> The real values are in `backend/esp32/src/main.cpp` (and git history) — they're
> intentionally **not** reproduced here so this checklist itself stays safe to share.

**Action:**
1. **Rotate** the backend `X-API-Key` on the server — assume the current one is burned.
2. Move secrets into a **gitignored `secrets.h`** (or PlatformIO `build_flags` from env), e.g.
   `#include "secrets.h"` with `secrets.h` in `esp32/.gitignore`.
3. If the repo will be made **public**, scrub history (`git filter-repo`) or accept that the
   old key/Wi-Fi password are exposed — rotating both is the minimum.

> The Wi-Fi here is a hotspot password, low risk, but the API key gated sensor ingestion.
